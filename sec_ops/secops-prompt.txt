3) MVP features (must-have)

Implement these end-to-end.

A) Alert ingestion + triage
	•	Provide an inbound webhook endpoint: /api/secops/alerts/ingest
	•	Accept JSON payloads from any tool; normalize into Security Alert records
	•	Compute dedupe_hash and correlation_key:
	•	Dedupe: same source + external_alert_id OR same indicators + title within 30 minutes
	•	Correlate: by asset_id + category + indicators across a rolling 24h window
	•	Auto-triage rules:
	•	If severity >= High AND asset criticality >= 4 → create incident (S2 or S1)
	•	If phishing category with user_id present → create incident (S3) and task user reset
	•	If repeated alert count > threshold in 1h → escalate priority
	•	UX:
	•	“Triage Queue” view with filters (severity, source, category, environment)
	•	Triage actions: Suppress, Close as Benign, Create Incident, Link to Incident, Run Playbook

B) Incident response lifecycle
	•	Incident states with transitions and required fields:
	•	New → In Progress → Contained → Eradicated → Recovered → Closed
	•	Enforce:
	•	Can’t move to Contained without containment_notes + affected_assets
	•	Can’t close without root_cause + lessons_learned
	•	Notifications:
	•	On creation: notify commander + assignee + SOC channel (placeholder integration)
	•	On SLA breach risk (e.g., 80% of time elapsed): notify assignee + commander
	•	Timeline:
	•	Every change writes an Audit Event and appends to incident timeline

C) Vulnerability management
	•	Inbound endpoint: /api/secops/vulns/ingest
	•	Normalize vuln scan results into Vulnerability Finding records
	•	Prioritization scoring:
	•	risk_score = severity_weight + (asset_criticality * factor) + exploitability_boost (if epss/cisa_kev flags provided)
	•	SLA assignment:
	•	Critical vulns due in 7 days on prod criticality>=4
	•	High vulns due in 30 days
	•	Flows:
	•	Assign remediation owner automatically based on asset owner_team
	•	Allow Accepted Risk with expiration + approval step by GRC role
	•	Verification step: mark Verified when scanner confirms fixed OR manual verification notes

D) Playbooks

Create at least 5 playbooks, each with steps and optional automation hooks:
	1.	Phishing Triage
	2.	Suspected Malware on Endpoint
	3.	Account Compromise / MFA Reset
	4.	Critical Vulnerability Patch Workflow
	5.	Data Exfiltration Investigation (lightweight)

Playbook runs should capture step results + collected artifacts.
