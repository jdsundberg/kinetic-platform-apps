Overview

ServiceProMax is the operational command center for professional services organizations. It provides visibility into:

  • Project health across the entire portfolio with multi-dimensional scoring
  • Delivery quality through embedded quality gates, audits, and corrective actions
  • Customer satisfaction with feedback tracking and follow-up management
  • Financial performance including budget variance, margin analysis, and burn rates
  • Resource utilization with consultant-level billable/non-billable tracking
  • Continuous improvement through lessons learned and delivery pattern analysis

Architecture

Browser (index.html)  -->  Base Server (port 3011)  -->  Kinetic Platform
   Single-page app           API proxy + custom          (first.kinetics.com)
   All CSS/JS inline         aggregation endpoints       Kapp: service-pro-max
                              server.mjs                  22 Forms, 100+ Indexes

Key design decisions:

  • Single HTML file with all CSS/JS inline — no build step, no external dependencies
  • System fonts only (-apple-system stack)
  • Server-side aggregation for dashboard KPIs via 9 custom API endpoints
  • Client-side pagination with 25-record limit per page
  • Auto-discovery by the base launcher — drop directory and restart

Quality Framework

Quality Gates (8 types)

  1. Intake Quality Gate — Validates project setup before approval
  2. Kickoff Readiness — Confirms team, timeline, milestones before start
  3. Delivery Health Review — Weekly delivery quality checks
  4. Milestone Readiness — Validates deliverables before milestone sign-off
  5. Customer Feedback Review — Processes customer satisfaction data
  6. Pre-Closeout Quality Gate — Validates all closeout checklist items
  7. Post-Project Retrospective — Captures lessons and best practices
  8. Delivery Audit — PMO-driven 8-dimension compliance review

Scoring Systems

  • Quality Reviews: 0-100 score with Pass/Fail/Partial checklist items
  • Delivery Audits: 8 dimensions (1-5 each), weighted to overall score
  • Customer Feedback: 1-5 scale for Delivery, Communication, Outcome
  • Project Health: 7 dimensions (Schedule, Budget, Scope, Staffing, Quality, Customer, Overall)

Maintenance Guide

Adding Seed Data

# Edit seed-data.json then run:
node setup-all.mjs --seed

Rebuilding Indexes

node setup-all.mjs  # Redefines fields/indexes and builds

Adding a New Form

  1. Add form definition to app.json with fields and indexes
  2. Run node setup-all.mjs to provision on the platform
  3. Add UI elements in index.html
  4. Add API endpoints in server.mjs if needed

Restarting the Server

kill $(lsof -ti:3011) && cd apps/base && node server.mjs &
