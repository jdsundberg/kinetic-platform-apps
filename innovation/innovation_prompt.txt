You are an enterprise product architect.

Design an "Innovation Application Intake System" for managing requests for new applications and application changes inside an organization.

The system should:

1. Allow users to submit:
   - Proposed Application Name
   - Business Justification (Why this app should exist)
   - Goals / Outcomes
   - Target Users
   - Expected Business Impact
   - Known Alternatives (if any)
   - Their own custom prompt content that will later be used to generate the application via AI
   - Attachments (optional)

2. After submission:
   - Route to the "Innovation Center" team queue
   - Set status = "Under Review"

3. Innovation Center capabilities:
   - Check if similar apps already exist
   - Evaluate overlap with current portfolio
   - Assess complexity, cost, risk
   - Request clarification from submitter
   - Approve / Reject / Merge / Redirect

4. Enable structured back-and-forth:
   - Comment threads
   - Clarification requests
   - Version tracking of the proposed prompt

5. When approved:
   - Generate a standardized `newapp.md` document
   - Include all structured information
   - Freeze approved prompt content
   - Mark status = "Ready for AI Build"

6. Include governance:
   - Audit trail
   - Approval timestamps
   - Innovation score (1-10)
   - Overlap risk score
   - Strategic alignment score

Design:
- Data model
- States
- Transitions
- Required fields
- Roles
- Governance controls
- Workflow diagram
- Example UI sections
- Escalation logic
- Rejection reasons

Keep it structured and implementation-ready.
