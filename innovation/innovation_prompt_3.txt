Governance Controls You Should Add
	•	Cannot approve if overlap risk > threshold
	•	Cannot move to “Ready for AI Build” without:
	•	Strategic alignment score
	•	Owner assigned
	•	Estimated complexity
	•	Limit revision cycles (e.g., max 3)
	•	Force structured rejection reasons








When reviewing, the team evaluates:
	1.	Does this already exist?
	2.	Can this be an enhancement?
	3.	Is this shadow IT?
	4.	Is this aligned to roadmap?
	5.	Is this automatable?
	6.	Is data sensitivity high?
	7.	Is AI generation appropriate?







This becomes:
	•	Application Portfolio Management
	•	Shadow IT governance
	•	AI-assisted app factory
	•	Structured innovation pipeline

Very powerful if done right.
